At Cloud Next ‘26, we <a href="https://workspace.google.com/blog/product-announcements/10-more-announcements-workspace-at-next-2026" target="_blank">announced</a> a new suite of agent tools for Google Workspace, including a Workspace MCP server, command line interface (CLI), and remote MCP integrations.&nbsp;<div><br /></div><div>Today, we are opening the <b><a href="https://developers.google.com/workspace/guides/configure-mcp-servers" target="_blank">Workspace MCP server</a></b> to public developer preview. As AI agents become a central part of how we work, they need a standardized way to access the right context and tools. The Workspace MCP server provides that bridge, allowing AI agents to safely tap into the richness of Workspace capabilities.</div><h3 style="text-align: left;">Workspace MCP server: Now in public developer preview</h3><div>The Workspace MCP server allows developers to access a suite of dedicated tools to power agents interacting with Workspace data:</div><div><br /></div><div><ul style="text-align: left;"><li><b>Gmail MCP: </b>Profile access, drafting, searching, and read/write capabilities.</li><li><b>Drive MCP: </b>File fetching, permissions management, listing, and uploading.</li><li><b>Calendar MCP: </b>Finding available times and managing events.</li><li><b>Chat MCP: </b>Finding conversations, searching messages, and reading/sending replies.</li><li><b>People dictionary MCP: </b>Managing user contacts and accessing profile information.</li></ul></div><h3 style="text-align: left;">Empower agents with a secure ecosystem</h3><div>As these agents scale, we are committed to maintaining a healthy ecosystem by addressing the unique risks associated with large-scale agentic actions, such as API abuse and unintended large-scale data egress. To foster innovation while managing the potential impact of high-volume data access, we are introducing a <a href="https://developers.google.com/workspace/tools-safety" target="_blank">standardized tiering model</a> for all agent tools inclusive of our APIs in Workspace.</div><div><br /></div><div>This model is designed to manage agent workflows at scale, providing a structure as application impact increases  The framework is intended to maintain the <b>agility of developers</b>:</div><div><br /></div><div><ul style="text-align: left;"><li><b>Empowering the community: </b>We expect <b>less than 1% of active Workspace developers</b> will need to move beyond our standard usage tiers. This means the vast majority of our developer community can continue to build and innovate without friction.</li><li><b>Scalability with confidence: </b>For those developers and agents that grow to serve millions of users, the model provides a clear, predictable path to scale safely, ensuring the performance and data integrity that enterprise customers expect.</li></ul></div><h4 style="text-align: left;">Getting started</h4><div><ul style="text-align: left;"><li><b>Developers: </b>Visit our developer documentation to <a href="https://developers.google.com/workspace/guides/configure-mcp-servers" target="_blank">learn more about the Workspace MCP server</a>. More information on the operational tiers is available in the <a href="https://developers.google.com/workspace/tools-safety" target="_blank">Workspace developer API tiering</a>.</li><li><b>Admins: </b>No action is required at this time. Access to Workspace APIs can be managed via the Admin console under <a href="https://knowledge.workspace.google.com/admin/apps/control-which-apps-access-google-workspace-data" target="_blank">Security &gt; API Controls</a>.</li></ul></div><h4 style="text-align: left;">Rollout pace</h4><div><ul style="text-align: left;"><li><b>Workspace MCP server: </b>Gradual rollout starting <b>May 1, 2026</b></li><li><b>Workspace API and MCP tiering: </b>Updated usage quotas for <b>new projects only</b> will roll out starting <b>May 1, 2026</b>. We'll notify owners of existing projects at least 60 days before any changes go into effect to ensure a smooth transition.</li></ul><h4 style="text-align: left;">Availability</h4></div><div><ul style="text-align: left;"><li>Available to all Google Workspace customers and developers</li></ul></div><h4 style="text-align: left;">Resources</h4><div><ul style="text-align: left;"><li><a href="https://developers.google.com/workspace" target="_blank">Google Workspace Developer Site</a></li><li>Google Workspace Developer Guide: <a href="https://developers.google.com/workspace/guides/configure-mcp-servers" target="_blank">Configure the Google Workspace MCP servers</a></li><li>Google Workspace Developer Documentation:&nbsp;<a href="https://developers.google.com/workspace/tools-safety" target="_blank">Google Workspace standardized model for agent tools and APIs</a></li></ul></div>
